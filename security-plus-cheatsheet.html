1.4App / Web Attacks
  • SQL Injection: INSERT/DROP em input não sanitizado
  • XSS (Reflected): Script injetado via URL/input, roda no browser da vítima
  • XSS (Stored): Script persistido no DB; atinge múltiplos usuários
  • CSRF: Forja request autenticado da vítima
  • IDOR: Acesso direto a objetos sem autorização
  • Directory Traversal: ../../../etc/passwd
  • Command Injection: ; rm -rf / em campos de entrada
  • XXE: XML External Entity; leitura de arquivos locais
  • SSRF: Força servidor a fazer req interna
  • Buffer Overflow: Escreve além do buffer; exec código
  • Race Condition: Explora janela de tempo entre check e use
  • API Abuse: Rate limit bypass, broken object level auth
1.5Crypto Attacks
  • Birthday: Colisão de hash; dois inputs = mesmo hash
  • Brute Force: Testa todas as combinações possíveis
  • Dictionary: Wordlist de senhas comuns
  • Rainbow Table: Hash pré-computado; derrotado por SALT
  • Pass-the-Hash: NTLM hash reutilizado sem crackear
  • Kerberoasting: Ticket do Kerberos offline brute-force
  • Downgrade: Força uso de protocolo fraco (SSL 3.0)
  • Padding Oracle: CBC mode; vaza info via erro de padding
  • POODLE: SSLv3 CBC padding oracle attack
  • BEAST: TLS 1.0 CBC chosen-plaintext
  • DROWN: SSLv2 afeta servidores TLS modernos
  • Harvest&Wait: Captura dados cifrados p/ decifrar futuramente

1.7Vuln Types
  • Zero-Day: Sem patch disponível; desconhecida do vendor
  • CVE: Common Vulnerabilities & Exposures (NVD)
  • CVSS: Score 0-10; Base/Temporal/Environmental
  • Misconfiguration: Default creds, open ports, perms erradas
  • Unpatched SW: EOL, legacy systems, patch lag
  • Weak Crypto: MD5, SHA1, DES, RC4, SSL/TLS 1.0
  • Open Permissions: World-writable files, overprivileged accounts
  • Supply Chain: SolarWinds, XZ Utils, npm package hijack
  • Memory Safety: Use-after-free, heap spray, int overflow
  • Side Channel: Spectre, Meltdown, timing attacks
  • Firmware: UEFI rootkits, BootHole, CosmicStrand

3.2Zero Trust & Segmentação
  • Zero Trust: "Never trust, always verify" — sem perímetro implícito
  • SASE: SD-WAN + SSE (SWG, CASB, ZTNA); cloud-native
  • Microsegmentation: Firewall por workload; E-W traffic control
  • ZTNA: Substitui VPN; acesso por identidade/contexto
  • SDP: Software-Defined Perimeter; oculta infra
  • DMZ: Zona desmilitarizada; servidores públicos isolados
  • VLAN: Segmentação L2 lógica; reduz broadcast domain
  • Air Gap: Isolamento físico total de rede
  • Jump Server: Bastion host; ponto único de acesso a infra
  • NAC: Network Access Control; postura antes de conectar
  • East-West: Tráfego interno entre workloads
  • North-South: Tráfego entrada/saída do datacenter

Identidade, Autenticação & Acesso
4.1Autenticação MFA
  • Type 1 — Know: Senha, PIN, perguntas secretas
  • Type 2 — Have: Token, smartcard, OTP, FIDO key
  • Type 3 — Are: Biometria: impressão, iris, voz, face
  • Somewhere: Geolocalização como fator adicional
  • TOTP: Time-based OTP (RFC 6238); Google Auth
  • HOTP: HMAC-based OTP; contador incremental
  • FIDO2/WebAuthn: Passkey; elimina senha; phishing-resistant
  • Push Notification: Approve/deny via app (Duo, MS Auth)
  • FAR: False Acceptance Rate (biometria)
  • FRR: False Rejection Rate (biometria)
  • CER: Crossover Error Rate; FAR=FRR (melhor métrica)

4.4Account Management
  • Provisioning: Criação de contas vinculada ao RH
  • Deprovisioning: Desativação imediata ao desligamento
  • Account Review: Revisão periódica de acessos (recertificação)
  • Dormant Accounts: Desativar após inatividade (90 dias)
  • Privilege Creep: Acúmulo de permissões; evitar
  • Service Accounts: Sem login interativo; senha rotacionada
  • Shared Accounts: Evitar; sem auditoria individual
  • Guest Accounts: Desabilitar por padrão
  • Admin Accounts: Usar apenas quando necessário
  • Account Lockout: Após N tentativas falhas (ex: 5)
  • Password Policy: Complexity, length, history, expiry
  • Time-of-Day: Restrição de horário de login

6.2Digital Forensics
  • Legal Hold: Preservação de dados para litígio
  • Order of Volatility: Mais volátil primeiro (RAM antes de HDD)
  • Disk Imaging: dd, FTK Imager; bit-by-bit copy
  • Write Blocker: Impede alteração da mídia original
  • Hash Verification: MD5/SHA256 da imagem = original
  • Chain of Custody: Quem tocou, quando, por quê
  • E-Discovery: Coleta e preservação de evidências digitais
  • Steganography: Dados ocultos em imagem/áudio
  • Metadata Analysis: EXIF, MFT timestamps, prefetch
  • Memory Forensics: Volatility Framework; processes, network
  • Network Forensics: PCAP; Wireshark; NetFlow
  • Log Forensics: Windows Event Logs; /var/log; syslog

9.3Backup & Redundância
  • Full Backup: Tudo; mais demorado; restauração rápida
  • Incremental: Mudanças desde último backup (qualquer)
  • Differential: Mudanças desde último FULL backup
  • 3-2-1 Rule: 3 cópias, 2 mídias distintas, 1 offsite
  • 3-2-1-1-0: + 1 offline/air-gapped + 0 erros de restore
  • Snapshot: Estado instantâneo; VM e storage
  • RAID 0: Striping; performance; sem redundância
  • RAID 1: Mirroring; 50% capacidade; tolera 1 falha
  • RAID 5: Striping+paridade; tolera 1 falha; min. 3 discos
  • RAID 6: 2 paridades; tolera 2 falhas; min. 4 discos
  • RAID 10: 1+0; mirroring+striping; alta perf+redundância
9.4Physical Security
  • Layered Defense: Perimeter → Building → Floor → Room → Device
  • Mantrap / Airlock: Dois portões; um abre por vez
  • Bollards: Postes anti-veículo; perimetral físico
  • Faraday Cage: Bloqueia EM; proteção TEMPEST
  • Biometric Door: Iris, impressão, retina para DCs
  • CCTV/DVR: Monitoramento; resolução e retenção
  • Motion Sensor: PIR (Passive Infrared), microwave
  • Lighting: Dissuasão; reduce dark spots
  • Guards: Deterrent + Detective + Response
  • Visitor Log: Chain of custody para acesso físico
  • Environmental: HVAC, fire suppression (FM-200, CO2), UPS
